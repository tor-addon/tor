Comment ça marche

La recherche interroge toutes vos sources en parallèle. Par défaut, elle attend que toutes les sources aient répondu avant d'afficher les résultats.

La condition d'arrêt vous permet de stopper plus tôt : dès qu'elle est satisfaite, la recherche s'arrête immédiatement — toutes les sources encore en cours sont annulées — et affiche ce qui a déjà été trouvé. Laissez le champ vide pour ne pas l'utiliser.

📊
Compteurs simples

Ces mots-clés représentent directement un nombre. Comparez-les à un chiffre avec >=, <=, >, <, =.

Mot-cléCe que ça compteExemple
streams Streams en cache disponibles — vérifiés et immédiatement jouables sur AllDebrid streams >= 5
torrent Streams de type torrent (Torznab, Torrent9, Library) disponibles en cache torrent >= 3
ddl Liens directs (Movix, Wawacity) — toujours disponibles ddl >= 1
Exemples
streams >= 5 # arrêter quand 5 streams sont en cache streams >= 3 or ddl >= 1 # 3 torrents en cache OU 1 lien direct trouvé ddl >= 1 # dès qu'un lien direct est trouvé
⏱️
Délai — timeout

delay représente le nombre de secondes écoulées depuis le début de la recherche. Utilisez-le pour forcer un arrêt après un certain temps.

Attention : Pour attendre N secondes, utilisez delay >= N — pas delay <= N.
delay <= 3 serait vrai dès la première seconde et stopperait immédiatement la recherche.
Exemples
delay >= 5 # arrêter après 5 secondes quoi qu'il arrive streams >= 5 or delay >= 10 # 5 en cache OU arrêt forcé après 10s streams >= 3 and delay >= 2 # au moins 2s ET 3 en cache (laisse les sources démarrer)
🔍
Filtrer par critère

Vous pouvez compter uniquement les streams qui correspondent à un critère précis : résolution, qualité ou langue. La syntaxe est :

Syntaxe générale
count( filtre( streams, 'critère' ) ) >= N # Exemples : count( resolution( streams, '>=1080p' ) ) >= 3 # 3 streams en 1080p ou plus count( lang( streams, 'fr' ) ) >= 2 # 2 streams en français

streams représente uniquement les streams déjà en cache sur AllDebrid — tous les filtres portent donc sur des streams immédiatement jouables.

Les filtres peuvent s'imbriquer — chaque filtre reçoit le résultat du précédent :

Imbrication de filtres
# Streams en français, en 1080p ou plus : count( resolution( lang( streams, 'fr' ), '>=1080p' ) ) >= 1
📺
Résolution

Filtre les streams selon leur résolution vidéo. Utilisez un opérateur pour définir une plage (>=1080p = Full HD et au-dessus).

480p→ 576p→ 720p→ 1080p→ 1440p→ 2160p / 4k
ValeurDescription
480pSD — qualité standard
720pHD — haute définition
1080pFull HD
1440p2K
2160p4K / UHD
4kIdentique à 2160p
Exemples
count(resolution(streams, '>=1080p')) >= 3 # 3 streams Full HD ou mieux count(resolution(streams, '>=2160p')) >= 1 # au moins 1 stream 4K count(resolution(streams, '1080p')) >= 2 # exactement 1080p (sans opérateur = égalité)
🎬
Qualité source

Filtre selon l'origine du fichier vidéo. Plus le rang est élevé, meilleure est la qualité audio/vidéo.

cam→ ts→ hdtv→ webrip→ web-dl→ bluray→ remux
ValeurDescription
camEnregistrement caméra en salle — très mauvais
tsTéléciné — avant la sortie officielle
hdtvEnregistrement TV HD
webripRip d'une plateforme de streaming, re-encodé
web-dlTéléchargement direct Netflix, Amazon, etc.
blurayRip depuis un disque Blu-ray
remuxCopie exacte du Blu-ray sans ré-encodage — qualité maximale
Exemples
count(quality(streams, '>=bluray')) >= 1 # 1 Bluray ou Remux en cache count(quality(streams, '>=web-dl')) >= 3 # 3 streams web-dl ou mieux
🌐
Langue

Filtre les streams selon leur langue audio. Utilisez le code à 2 lettres de la langue.

Français : Le code 'fr' reconnaît automatiquement toutes les variantes — VF, VFF, VFQ, Truefrench, Québécois. Vous n'avez pas à les spécifier séparément.
CodeLangue
'fr'Français (VF, VFF, VFQ, Truefrench inclus)
'en'Anglais
'es'Espagnol
'de'Allemand
'it'Italien
'pt'Portugais
'ja'Japonais
'ko'Coréen
Exemples
count(lang(streams, 'fr')) >= 1 # au moins 1 stream français count(lang(streams, 'en')) >= 5 # 5 streams en anglais

Exemples — Arrêt rapide
Basiquevitesse
streams >= 5
Arrêter dès que 5 streams sont disponibles en cache. Simple et efficace pour du contenu populaire.
Timeout fixevitesse
delay >= 8
Collecter pendant exactement 8 secondes puis afficher tout ce qui a été trouvé.
Résultats rapides ou timeoutvitesse
streams >= 5 or delay >= 10
Arrêter dès 5 en cache, ou dans tous les cas après 10 secondes. Bon équilibre vitesse/résultats.
Minimum avant arrêtvitesse
streams >= 5 and delay >= 2
Attendre au moins 2 secondes avant d'arrêter, même si les 5 résultats arrivent très vite. Laisse le temps aux sources lentes de répondre.
Priorité aux liens directsvitesse
ddl >= 1 or delay >= 12
Arrêter dès qu'un lien direct (Movix/Wawacity) est trouvé, sinon timeout à 12s.

Exemples — Combinés
FR ET qualitécombiné
(count(lang(streams, 'fr')) >= 1 and count(resolution(streams, '>=1080p')) >= 2) or delay >= 12
1 stream FR en cache ET 2 streams 1080p — sinon timeout à 12s.
Plusieurs conditions alternativescombiné

Séries — efficacecombiné

Bon réglage pour les séries : 2 streams HD en cache suffisent, timeout 10s.
